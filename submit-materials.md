# Submitting Digital Materials for the ABSEL Store
## Draft by Precha Thavikulwat

The Association for Business Simulation and Experiential Learning (ABSEL) keeps digital materials associated with its [conference proceedings (BKL)](https://absel-ojs-ttu.tdl.org/absel/index) and [journal (JELB)](https://jelb.org/) in the [ABSEL Store](https://github.com/absel-store). Each set of related materials is kept in its own [GitHub](https://github.com/) repository. To facilitate the review and placement of items into the ABSEL Store, applicants with materials for the ABSEL Store must have an account on GitHub, which is free at the basic level.

To pass review, every item in the candidate repository must be accessible *without formalities*, that is, without requiring the user to relinquish privacy by contacting someone or entering a real identity. This privacy-protecting condition is generally met in files that are not password protected, but it also can be met in a password-protected file provided the required password is given in an accompanying file that is not itself password protected. The user must not be required to call, email, or otherwise reach out to receive the password. Likewise, if a repository item item is an application or a conduit to a website that requires registration, the user must be able to register with false credentials that protect privacy.

The process of submitting digital materials for review involves three steps: First, create the candidate repository on GitHub. Second, set its attribute to public-template, thereby enabling the candidate repository to be copied to the ABSEL Store in a single action and avoiding the tedious process of downloading and uploading files. Third, send an email requesting review.

### Creating a Repository
To create a repository, on the GitHub Homepage/Dashboard, click the **New** button on the left side of the page. In the **Create a new repository** form that appears, enter the required **Repository name** of your choosing, such as *stuff-for-absel*. Then accept all defaults and click the **Create Repository** button.
