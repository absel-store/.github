# Submitting Digital Materials for the ABSEL Store
## Draft by Precha Thavikulwat, 16 July 2026

The Association for Business Simulation and Experiential Learning (ABSEL) keeps digital materials associated with its [conference proceedings (BKL)](https://absel-ojs-ttu.tdl.org/absel/index) and [journal (JELB)](https://jelb.org/) in the [ABSEL Store](https://github.com/absel-store). Each set of related materials is kept in its own [GitHub](https://github.com/) repository. To facilitate the review and placement of items into the ABSEL Store, applicants with materials for the ABSEL Store must have an account on GitHub, which is free at the basic level.

To pass review, every item in the candidate repository must be accessible *without formalities*, that is, without requiring the user to relinquish privacy by contacting someone or entering a real identity. This privacy-protecting condition is generally met by files that are not password protected, but it also can be met by a password-protected file when the required password is given in an accompanying file that is not itself password protected. Moreover, the user must not be required to call, email, or otherwise reach out to receive the password. Likewise, if a repository item item is an application or a conduit to a website that requires registration, the user must be able to register with false credentials that protect privacy.

The process of submitting digital materials for review involves three steps: First, create the candidate repository on GitHub. Second, set its attribute to public-template, thereby enabling the candidate repository to be copied to the ABSEL Store in a single action and avoiding the tedious process of downloading and uploading files. Third, send an email requesting review.

### Creating a Repository
To create a repository, on the GitHub Homepage/Dashboard, click the **New** button on the left side of the page. In the **Create a new repository** form that appears, enter the required **Repository name** of your choosing, such as *stuff-for-absel*. Then, accepting all defaults, click the **Create Repository** button.

Repository created, the created repository’s opening page appears, headed by the repository’s name. When ready to populate the repository with files, click **uploading an existing file**, a link within the **Quick setup** area of the page. Then either use the mouse to drag-and-drop the files from your computer to the drag-files-here area of the page that appears or click the **choose your files** link to browse for and select the files. Upload successful, the names of the files that have been uploaded will be listed above the Commit changes area. Click the **Commit changes** button below the area after all files selected have been uploaded. Action successful, the repository's opening page changes to show its menu below its heading, with the leftmost item, **Code**, selected.

### Setting the Template Attribute
The default attribute of the created repository is public but not a template. If the repository is to be reviewed for the ABSEL Store, its attribute must be public and a template. To set the repository’s template attribute, select its menu's rightmost item, **Settings**, appearing just below the heading of its webpage. 

On the Settings webpage, in the **General** area, click to check the box for **Template repository**. A check mark appearing in response at the immediate right of the label affirms a successful change.

### Email Requesting Review
Email the review request to the address shown just below the [ABSEL Store’s webpage](https://github.com/absel-store). In the email, list the items in your candidate repository, each item accompanied by a one-sentence description of its content. Also specify the full internet address of the repository. For example, for a repository named *stuff-for-absel* created on the homepage of the GitHub account named *johndoe*, the repository’s address will be *https://github.com/johndoe/stuff-for-absel*.
