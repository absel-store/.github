# How to Contribute to this Website

## Drafted by Precha Thavikulwat, 7/21/2026

GitHub tracks contributions, which means that contributors must first be logged in to a GitHub account, free at the basic level. Contributions can be made to wikis, discussions, and the contents of repositories.

  Wikis are public paged documents that contributors can edit directly. ABSEL's wiki is attached to the .github repository. The standard path to the wiki is from the **Repositories** menu item of [ABSEL Store](https://github.com/absel-store) to [.github](https://github.com/absel-store/.github). To contribute, bring up [ABSEL Store wiki](https://github.com/absel-store/.github/wiki). Then click the **New page** button at the top right of the page to add a page, or click one of the links of the sidebar on the right to bring up an existing page.
  
  Discussions are public chats, also are attached to the .github repository, but the standard path to them, appearing as the **Discussions** menu item on the [ABSEL Store webpage](https://github.com/absel-store), is more direct. To contribute, bring up [ABSEL Store discussions](https://github.com/orgs/absel-store/discussions) The click the **New discussion** button at the top right of the page to start a new discussion, or click one of the listed discussions to join it. Type your comment into the **Add a comment** edit box, then click **Commit** to submit it. Submission accepted, the comment appears, to which others might reply. To change either comment or reply, click the three-dot (...) menu to the right of the item, and select the desired change: edit, hide, or delete.

  Repositories are containers for mission-critical content. Everyone with a GitHub account can create and edit the contents of a personal repository, but only privileged members can do likewise for an organizational repository. If you are not privileged to create an ABSEL Store repository, but you have products that you would like to contribute nonetheless, please read [Submitting Digital Materials for the ABSEL Store](https://github.com/absel-store/.github/blob/main/submit-materials.md).  
  
  To create a repository, from the [ABSEL Store](https://github.com/absel-store), click the **Repositories** menu item. A list of the store's repositories will appear. If you are privileged to create an ABSEL Store repository, a **New repository** button will appear on upper right of the area. Click that button. Then fill in the repository creation form that appears and click the **Create repository** on the bottom of the form. Repository created, you can proceed to set it up by clicking, in the **Quick setup** box that appears, the link to *creating a new file* or *uploading an existing file* to put files into the repository. 

   Editing a file of a repository at the request of one who did not create the repository, however, requries action by the requester asking for the change, the reviewers designated to approve the change, and the assignees designated to execute the change. The requester begins the process by taking the following steps:
  1. *Fork* the repository. Forking is action that creates a copy of the target repository for the requester's account that can subsequently be updated with changes to the target repository.
  2. Create an offshoot _branch_ of the repository's _main_ branch.
  3. Edit the file.
  4. Commit, that is, save the changes to the offshoot _branch_.
  5. Send a *pull request* to collaborators. Essentially, collaborators are asked to pull down the offshoot branch for examination.
     
Upon approval of the requested changes, the offshoot branch is _merged_ to the _main_ branch, either automatically following a rule or at the discretion of an assignee.
